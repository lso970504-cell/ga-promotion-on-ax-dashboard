# AGENTS.md

## 프로젝트 목적

이 워크스페이스는 `GA 프로모션 ON AX` 운영 대시보드와 안내장 템플릿을 관리한다.  
핵심 목적은 본사담당자가 월별 프로모션 데이터를 업로드하고, 영업관리자가 현황 조회와 다운로드를 쉽게 할 수 있도록 유지하는 것이다.

## 현재 기준 파일

- 메인 화면: `C:\Users\user\Documents\코덱스_day6\20260617.html`
- 보조 화면: `C:\Users\user\Documents\코덱스_day6\index.html`
- 단독 템플릿: `C:\Users\user\Documents\코덱스_day6\promo-template.html`
- 대시보드용 템플릿: `C:\Users\user\Documents\코덱스_day6\template-package\promo-template.html`
- 템플릿 데이터 매핑: `C:\Users\user\Documents\코덱스_day6\promo-template-data.browser.js`

## 작업 우선순위

1. 사용자 화면 문구/배치 수정
2. 프로모션 현황 조회 정합성 수정
3. 관리자 업로드/승인 흐름 수정
4. 안내장 템플릿 출력 수정
5. 배포/문서 정리

## 반드시 지킬 규칙

### 1. 메인 기준 파일

- 대시보드 수정은 기본적으로 `20260617.html` 기준으로 작업한다.
- 사용자가 `index.html`을 따로 지정하지 않으면 최신 작업 기준은 `20260617.html`로 본다.

### 2. 템플릿 동시 수정

- 템플릿 동작을 수정할 때는 아래 두 파일을 함께 본다.
  - `promo-template.html`
  - `template-package/promo-template.html`

### 3. 사용자 요청 반영 방식

- 사용자는 문구, 배치, 정렬, 버튼 위치를 자주 세부 수정한다.
- 큰 리디자인보다 작은 수정 누적으로 진행하는 편이 맞다.
- 제목만 바꾸지 말고 관련 버튼명, 표 헤더, 저장 파일명도 같이 본다.

### 4. 데이터 해석 규칙

- `지역단`은 조직 시트와 데이터 시트의 관리지점 기준으로 맞춘다.
- `GA지사`는 데이터의 `GA명/GA지사` 기준을 우선한다.
- `확정/미확정/오류`는 다운로드 가능 여부와 연결된다.
- 미확정은 `프로모션 수치 미입력` 의미를 기본값으로 본다.

### 5. 권한 규칙

- 본사담당자(`hq`)만 관리자페이지를 본다.
- 영업관리자(`branch`)는 관리자 섹션을 보지 않는다.
- 로그인은 `사번 5자리 = 비밀번호 5자리` 규칙이다.

## 작업 시 주의사항

### 화면 수정

- 표 너비, 버튼 폭, 긴 GA지사명 줄바꿈을 항상 같이 확인한다.
- 검색 필터와 테이블 헤더는 한 줄 정렬이 중요하다.
- 사용자는 “보기 편한 화면”을 매우 중요하게 본다.

### 업로드 수정

- 업로드 후 즉시 반영 알럿이 보여야 한다.
- 월별 데이터는 덮어쓰기만 하지 말고 월 기준 상태를 유지해야 한다.
- 로컬 저장 상태와 기본 샘플 데이터가 충돌하지 않는지 확인한다.

### 템플릿 수정

- 사용자는 표준 템플릿과 AI 자동생성 방향을 오가며 비교한다.
- 템플릿 레이아웃을 바꿀 때는 하위 블록 숨김/빈값 처리 여부를 같이 점검한다.
