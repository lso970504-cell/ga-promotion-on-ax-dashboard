# 보험 GA 프로모션 안내장 AI 자동생성 규칙서

## 1. 목적

이 문서는 `월별로 달라지는 보험 GA 프로모션 구조`를 기준으로,  
고정 표준템플릿 대신 `AI 기반 안내장 이미지 자동생성`을 하기 위한 운영 규칙을 정리한다.

핵심 목표는 아래 3가지다.

1. 업로드된 엑셀 데이터를 자동 해석한다.
2. 지사별 프로모션 안내장 이미지를 자동 생성한다.
3. 본사담당자가 미리보기와 승인 후 최종 이미지를 배포한다.

---

## 2. 기본 방향

- 결과물의 중심은 `안내장 이미지`다.
- 이미지는 `보험사 현장 배포용 전단형 구조`를 따른다.
- 월마다 시상 구조, 카드 수, 표 구성이 달라질 수 있음을 전제로 한다.
- 따라서 `고정 템플릿에 값만 주입`하는 방식이 아니라 `데이터 구조를 먼저 해석하고 레이아웃을 조합`하는 방식으로 생성한다.

---

## 3. 디자인 원칙

### 공통 감성

- 금융회사 공식 홍보물 수준
- 신뢰감 있는 블루 메인 컬러
- 옐로 또는 골드 포인트 컬러
- 정보가 많아도 한눈에 읽히는 구조
- A4 세로형 전단 레이아웃

### 절대 유지할 구조

1. 상단 헤드라인은 강하게 보인다.
2. 핵심 KPI는 가장 크게 보인다.
3. 시상 구조는 하단에 카드/표 혼합형으로 정리한다.
4. 유의사항과 참여조건은 최하단에 작지만 읽히게 둔다.

### 고정하지 않는 요소

- 시상구분 개수
- 시상 블록 수
- 표의 열 개수
- KPI 강조 수치 개수
- 대리점별 강조 문구

---

## 4. 이미지 레이아웃 규칙

### 상단 영역

- 프로모션명
- 기준월
- 대상상품
- 대리점명 또는 사업부명
- 운영기간

표현 원칙:

- 제목은 가장 크게 배치한다.
- 기준월은 제목 옆 또는 상단 띠 형태로 강조한다.
- 대리점명은 우측 상단 배지형 또는 박스형으로 둘 수 있다.

### 중상단 KPI 영역

- 최대 목표율
- 최대 달성률
- 대표 시상금
- 핵심 KPI 1~3개

표현 원칙:

- 가장 눈에 띄는 수치만 크게 쓴다.
- 예: `500%`, `300% + 200%`, `최대 500만원`
- 수치가 여러 개면 한 줄 KPI 바 또는 2~3칸 KPI 박스로 구성한다.

### 중단 시상 핵심 영역

- 가장 강조해야 할 시상 1~2개를 별도 띠 또는 강조 박스로 배치한다.
- 예: `기본 가동 시상`, `정규 시상`, `연속 가동 시상`

### 하단 상세 영역

시상구분 수에 따라 자동 변경한다.

- 1~2개: 카드형 중심
- 3~4개: 카드 + 표 혼합형
- 5개 이상: 표 중심 + 핵심 카드 보조형

각 블록에는 아래를 넣는다.

- 시상명
- 달성조건
- 시상금액
- 운영기간 또는 회차
- 특이사항

### 최하단 영역

- 참여대상
- 유의사항
- 예외사항
- 비고

---

## 5. 월별 자동 변경 규칙

이미지는 `매월 달라질 수 있음`을 정상 상태로 본다.

### 바뀌어도 되는 것

- 블록 수
- 표 크기
- 헤드라인 길이
- KPI 배치 수
- 카드형/표형 비중

### 바뀌지 않아야 하는 것

- 금융회사 전단형 느낌
- 상단 강한 제목 구조
- KPI 우선 시선 흐름
- 하단 상세 정보 정리 구조
- 가독성 우선 원칙

---

## 6. 데이터 해석 규칙

엑셀 업로드 후 AI는 아래 순서로 데이터를 정리한다.

1. 프로모션 기본정보 해석
2. KPI 후보 추출
3. 시상구분 자동 그룹화
4. 시상조건 및 금액 정리
5. 유의사항/예외사항 분리

### 중요 원칙

- 시상구분은 고정 사전 정의하지 않는다.
- 엑셀에 실제로 있는 값만 읽어 그룹화한다.
- 누락값은 추정하지 않는다.
- 서로 충돌하는 값은 `확인 필요`로 표시한다.

---

## 7. 생성 단위 규칙

현재 운영 기준은 `지사별 또는 행 기준 생성` 중 업무 규칙에 따라 선택 가능하다.

권장 기본값:

- 대시보드에서 사용자가 선택한 지사 기준으로 1건 생성
- 같은 지사 안에 여러 시상구조가 있으면 1장 안에서 통합 배치

예외:

- 시상구조가 너무 많으면 1장 안에서 표 중심으로 축약
- 데이터 누락이 있으면 `미확정`
- 구조 충돌이 심하면 `오류`

---

## 8. 승인 규칙

### 확정

- 필수 수치가 모두 있음
- 시상구조가 해석 가능함
- 이미지 생성 가능함

### 미확정

- 프로모션 수치 누락
- 필수 조건 문구 부족
- 관리자가 추가 확인 필요

### 오류

- 지사 매핑 실패
- 데이터 구조 충돌
- 시상조건/금액 해석 실패

### 운영 원칙

- 본사담당자가 관리자페이지에서 미리보기 확인
- 승인 전에는 초안 미리보기만 허용
- 승인 후에만 최종 이미지 다운로드 허용

---

## 9. 대시보드 연계 규칙

### 관리자페이지 흐름

1. 프로모션 엑셀 업로드
2. AI 안내장 생성
3. 행별 또는 지사별 미리보기
4. 전체 승인
5. 확정건 다운로드 허용

### 사용자 화면 흐름

- 지역단 / 지점 / GA명 / 상태 기준 조회
- 확정건만 이미지 다운로드 가능
- 미확정/오류는 다운로드 비활성화

---

## 10. AI 생성 프롬프트 핵심 규칙

프롬프트에는 반드시 아래 내용을 포함한다.

- 금융회사 공식 프로모션 전단 스타일
- 블루 메인 컬러와 골드/옐로 포인트
- A4 세로형
- 상단 제목 + 중단 KPI + 하단 시상구조
- 시상구분 수에 따라 카드/표 자동 조합
- 가독성 우선

추가 규칙:

- 지나치게 광고성으로 만들지 않는다.
- 화려함보다 신뢰감과 정리가 중요하다.
- 텍스트가 많아도 블록 구분이 명확해야 한다.

---

## 11. 향후 확장

향후 아래까지 연결 가능하다.

- 생성된 이미지 기반 PPT 1장 자동생성
- 관리자 수정문구 반영
- 월별 이미지 스타일 A/B 선택
- 상품군별 별도 레이아웃 규칙
- 지사별 우선 강조 KPI 자동선정

---

## 12. 결론

이 방식은 `표준 템플릿 고정형`보다 `월별 데이터 구조가 달라지는 보험 GA 프로모션 운영`에 더 적합하다.

따라서 향후 시스템 방향은 아래처럼 가져간다.

1. 엑셀 자동 해석
2. AI 기반 레이아웃 조합
3. 안내장 이미지 생성
4. 본사담당자 승인
5. 확정건 배포

즉, `이미지 구조는 매월 달라질 수 있지만, 생성 원칙은 고정`하는 방식으로 운영한다.
