# GA 프로모션 안내장 두번째 시트 메타정보 입력 표준안

## 1. 목적

이 문서는 `GA 프로모션 안내장 자동생성`을 위해  
업로드 엑셀의 `두번째 시트`에서 어떤 값을 어떤 형식으로 입력해야 하는지 정리한 운영 표준안이다.

핵심 목표는 아래 4가지다.

1. 엑셀 업로드 시 시스템이 `항상 두번째 시트`만 읽도록 고정한다.
2. 제목, 대상상품, 시상기간, 유의사항을 `엑셀에서 직접 관리`할 수 있게 한다.
3. 프로모션 블록은 `매월 추가 또는 삭제 가능`하게 만든다.
4. 안내장 이미지는 `하늘색 파스텔톤 표준안내장` 스타일로 자동 생성한다.

---

## 2. 기본 운영 원칙

- 업로드 기준 시트는 `항상 workbook.SheetNames[1]`, 즉 두번째 시트다.
- 제목은 고정 문구가 아니라 `엑셀 기준월 값`으로 자동 변경한다.
- 블록 제목은 시스템이 임의로 만들지 않고 `엑셀에 있는 실제 블록명 그대로` 사용한다.
- 월별로 블록이 줄거나 늘어나는 것은 정상 상태로 본다.
- 하단 `적용여부` 블록은 표준안내장 구조에서 제외한다.

---

## 3. 메타정보 입력 위치 표준

두번째 시트 상단에 아래처럼 `항목 / 값` 2열 구조를 둔다.

### 권장 입력 위치

- `A1`: 항목
- `B1`: 값

- `A2`: 기준월
- `B2`: 2026.06

- `A3`: 대상상품
- `B3`: 행복종신/플러스원/암치료비플러스/순수3종

- `A4`: 시상기간
- `B4`: 6/1 ~ 6/28

- `A5`: 유의사항1
- `B5`: 세부 지급기준 및 적용조건은 본사 최종 공지 기준으로 운영됩니다.

- `A6`: 유의사항2
- `B6`: 프로모션별 중복 적용 여부는 항목별 기준을 확인해 주세요.

- `A7`: 유의사항3
- `B7`: 최종 확정 전 안내장은 변경될 수 있습니다.

### 운영 이유

- 셀 주소를 외우는 방식보다 `항목명 + 값` 형태가 관리하기 쉽다.
- 담당자가 바뀌어도 엑셀만 보면 바로 입력 위치를 이해할 수 있다.
- 월별로 문구 수정이 있어도 별도 코드 수정 없이 운영 가능하다.

---

## 4. 제목 생성 규칙

### 입력값

- `기준월`: `2026.06` 형식 권장

### 출력값

- 제목 표시: `26.6월 프로모션`

### 규칙

- 제목은 `월만 자동 변경`한다.
- GA명은 제목에 붙이지 않는다.
- GA명은 우측 상단 박스 또는 별도 식별 영역에 표시한다.

예시:

- 기준월 `2026.06` → `26.6월 프로모션`
- 기준월 `2026.07` → `26.7월 프로모션`

---

## 5. 상단 보조 문구 규칙

### 입력값

- `대상상품`

### 출력값

- 상단 제목 아래 보조 문구로 표시한다.

예시:

- 입력: `행복종신/플러스원/암치료비플러스/순수3종`
- 출력: `대상상품, 행복종신/플러스원/암치료비플러스/순수3종`

### 규칙

- 보조 문구는 제목보다 작은 크기로 표시한다.
- 상품군은 엑셀 값을 그대로 사용하되, 너무 길면 줄바꿈을 허용한다.

---

## 6. 유의사항 입력 규칙

유의사항은 하단에 자동 출력하되, 엑셀에서 직접 관리한다.

### 입력 방식

- `유의사항1`
- `유의사항2`
- `유의사항3`

필요시 이후 확장:

- `유의사항4`
- `유의사항5`

### 출력 방식

- 하단에 작은 글씨 2~4줄 정도로 정리
- 값이 비어 있으면 해당 줄은 출력하지 않음
- 모두 비어 있으면 기본 문구 사용 가능

### 기본 문구 예시

```text
※ 세부 지급기준 및 적용조건은 본사 최종 공지 기준으로 운영됩니다.
※ 프로모션별 중복 적용 여부는 항목별 기준을 확인해 주세요.
※ 최종 확정 전 안내장은 변경될 수 있습니다.
```

### 운영 원칙

- 유의사항은 한 셀에 길게 몰아쓰지 않는다.
- 줄 단위로 나누어 입력한다.
- 이미지 하단 공간이 좁을 경우 3줄까지만 우선 노출한다.

---

## 7. 블록 자동생성 규칙

### 기준

- 블록명은 `엑셀 두번째 시트에 실제 있는 제목 그대로` 사용한다.
- 시스템이 임의로 `기타시상` 같은 새 이름을 붙이지 않는다.
- 현재 예시는 아래 블록명을 기준으로 본다.

예시 블록:

- `정규시상`
- `기본가동프로모션`
- `5~6월 브릿지`
- `6~7월 브릿지`
- `Gold 시상`

### 핵심 원칙

- 어떤 달에는 `Gold 시상`이 있을 수 있다.
- 어떤 달에는 `Gold 시상`이 없을 수 있다.
- 어떤 달에는 새 블록이 추가될 수 있다.
- 따라서 표준안내장은 `고정 4블록` 구조가 아니라 `실제 존재하는 블록만 자동 생성` 구조여야 한다.

### 생성 방식

1. 두번째 시트에서 블록 제목을 읽는다.
2. 블록 제목 순서를 유지한다.
3. 각 블록별 표 구조를 만든다.
4. 없는 블록은 숨긴다.
5. 새 블록은 같은 스타일로 추가 생성한다.

---

## 8. 디자인 기준

### 스킨 방향

- 전체 스킨은 `하늘색 파스텔톤`
- 기존 진한 파랑 배경보다 더 편안하고 부드러운 톤
- 금융회사 공식 안내장처럼 깔끔한 인상 유지

### 유지 요소

- 삼성생명 계열 블루 포인트
- 노란 강조 컬러 일부 사용
- 상단 헤더 + 표 중심 블록 구조
- A4 세로형 안내장

### 제외 요소

- KPI 카드형 강조
- 하단 적용여부 블록
- 과한 광고형 장식
- 활동물품 이미지

---

## 9. 시스템 반영 규칙

### 업로드

- 관리자가 업로드하는 기준 파일은 `엑셀 전체 파일`이지만,
  시스템은 `항상 두번째 시트`만 읽는다.

### 제목 생성

- `기준월` 메타정보를 읽어 `26.6월 프로모션` 형식으로 변환

### 하단 문구 생성

- `유의사항1~3`을 읽어 하단에 줄 단위로 출력

### 블록 생성

- 실제 존재하는 블록만 렌더링
- 블록 수 증감에 맞춰 안내장 길이와 표 개수를 자동 조정

---

## 10. 완료 기준

이 표준안이 제대로 반영되었다고 볼 수 있는 조건은 아래와 같다.

1. 사용자가 두번째 시트 상단 메타정보만 수정해도 제목과 유의사항이 바뀐다.
2. 기준월이 달라지면 제목도 자동으로 바뀐다.
3. 블록명은 엑셀에 적힌 실제 이름 그대로 나온다.
4. 어떤 달에는 블록이 늘고, 어떤 달에는 줄어도 이미지가 깨지지 않는다.
5. 표준안내장 스타일은 `하늘색 파스텔톤`으로 유지된다.
