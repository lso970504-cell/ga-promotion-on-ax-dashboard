# GA 프로모션 관리자 엑셀 업로드 화면 와이어프레임안

## 1. 목적

이 문서는 `본사담당자 관리자페이지`에서 사용하는  
`엑셀 업로드 화면 레이아웃`과 `화면 문구 표준안`을 정리한 것이다.

핵심 목표는 아래와 같다.

1. 사용자가 업로드 후 무엇이 반영되는지 바로 이해할 수 있어야 한다.
2. 시스템이 `두번째 시트만 읽는다`는 점이 화면에서 명확해야 한다.
3. 제목, 유의사항, 블록 구성이 어떻게 잡히는지 업로드 직후 미리 확인 가능해야 한다.
4. `업로드 → 미리보기 → 전체 승인` 흐름이 한 화면에서 이해되어야 한다.

---

## 2. 전체 화면 구조

관리자 업로드 화면은 4개 구역으로 나눈다.

1. 엑셀 업로드 영역
2. 메타정보 미리보기 영역
3. 블록 인식 결과 영역
4. 검증 결과 및 실행 버튼 영역

권장 배치는 아래와 같다.

- 1행 전체: `엑셀 업로드`
- 2행 좌측: `메타정보 미리보기`
- 2행 우측: `블록 인식 결과`
- 3행 전체: `검증 결과`
- 하단: `표준안내장 미리보기 생성 / 전체 승인`

---

## 3. 1행 업로드 영역

### 섹션 제목

- `엑셀 업로드`

### 본문 문구

- `프로모션 운영 엑셀을 업로드하면 두번째 시트만 자동 반영됩니다.`

### 보조 문구

- `기준월, 대상상품, 시상기간, 유의사항은 두번째 시트 상단 메타정보를 읽습니다.`
- `표준안내장 블록은 두번째 시트에 있는 실제 블록명 기준으로 자동 구성됩니다.`

### 화면 요소

- 큰 업로드 박스
  - 문구: `엑셀 파일을 여기에 끌어놓거나 선택해 주세요`
- 버튼
  - `파일 선택`

### 업로드 완료 후 표시 정보

- `파일명`
- `업로드 일시`
- `업로드한 담당자`

예시:

- `파일명: 6월 프로모션 총괄판(260529)_공지5.xlsx`
- `업로드 일시: 2026-06-17 10:30`
- `업로드 담당자: 본사담당자`

---

## 4. 2행 좌측 메타정보 미리보기

### 섹션 제목

- `메타정보 미리보기`

### 설명 문구

- `두번째 시트 상단 입력값을 기준으로 안내장 제목과 하단 문구가 생성됩니다.`

### 표시 항목

- `기준월`
- `대상상품`
- `시상기간`
- `유의사항1`
- `유의사항2`
- `유의사항3`

### 함께 보여줄 자동 변환 결과

- `생성 제목`

예시:

- `기준월: 2026.06`
- `생성 제목: 26.6월 프로모션`
- `대상상품: 행복종신/플러스원/암치료비플러스/순수3종`
- `시상기간: 6/1 ~ 6/28`

### 표시 방식

- 표 또는 읽기 전용 카드 형식
- 수정은 여기서 하지 않고 엑셀 수정 후 재업로드 방식 유지

---

## 5. 2행 우측 블록 인식 결과

### 섹션 제목

- `블록 인식 결과`

### 설명 문구

- `표준안내장은 두번째 시트에서 실제 인식된 블록만 자동으로 구성됩니다.`

### 표시 항목

- `인식된 블록 수`
- `블록 목록`

예시:

- `총 5개 블록 인식`
- `정규시상`
- `기본가동프로모션`
- `5~6월 브릿지`
- `6~7월 브릿지`
- `Gold 시상`

### 표시 방식

- 카드형 또는 리스트형
- 새 블록이 있으면 그대로 추가 표시
- 없는 블록은 표시하지 않음

### 보조 문구

- `월별로 블록은 추가되거나 제외될 수 있습니다.`

---

## 6. 3행 검증 결과 영역

### 섹션 제목

- `업로드 검증 결과`

### 설명 문구

- `업로드한 파일이 표준안내장 생성 조건을 충족하는지 확인합니다.`

### 상태 요약 카드

- `정상`
- `미확정`
- `오류`

### 각 상태 의미

- `정상`: 표준안내장 생성 가능
- `미확정`: 필수 값 일부 확인 필요
- `오류`: 생성 불가 또는 시트 구조 확인 필요

### 예시 메시지

- `메타정보 확인 완료`
- `블록 5개 인식 완료`
- `표준안내장 생성 가능`

오류 예시:

- `기준월 값이 없습니다.`
- `두번째 시트 상단 메타정보를 확인해 주세요.`
- `블록 제목을 인식하지 못했습니다.`

---

## 7. 하단 실행 버튼 영역

권장 버튼 순서는 아래와 같다.

1. `업로드 반영`
2. `표준안내장 미리보기 생성`
3. `전체 승인`

### 버튼 설명 문구

- `업로드 반영`: 두번째 시트 데이터를 읽어 대시보드에 반영
- `표준안내장 미리보기 생성`: 업로드 데이터를 기준으로 시안 생성
- `전체 승인`: 최종 확정 처리 후 사용자 다운로드 허용

### 버튼 활성화 규칙

- 업로드 전: 모두 비활성화
- 업로드 후 검증 성공: `업로드 반영`, `미리보기 생성` 활성화
- 미리보기 생성 후: `전체 승인` 활성화

---

## 8. 추천 화면 문구 모음

### 상단 안내문

- `프로모션 운영 엑셀 업로드`
- `두번째 시트 기준으로 표준안내장 데이터가 자동 반영됩니다.`

### 업로드 박스 안 문구

- `엑셀 파일을 끌어놓거나 선택해 주세요`
- `업로드 시 두번째 시트만 읽습니다`

### 메타정보 안내문

- `기준월과 유의사항은 엑셀 상단 메타정보 기준으로 자동 반영됩니다.`

### 블록 안내문

- `블록명은 엑셀에 적힌 실제 제목 그대로 사용됩니다.`

### 검증 안내문

- `미리보기 생성 전 업로드 상태를 확인해 주세요.`

---

## 9. 운영 원칙

- 업로드 후 화면에서 직접 값 수정은 하지 않는다.
- 값 수정이 필요하면 엑셀 수정 후 재업로드한다.
- 화면은 `무엇이 읽혔는지`를 보여주는 데 집중한다.
- 본사담당자가 업로드와 전체 승인을 담당한다.

---

## 10. 완료 기준

이 화면이 잘 설계되었다고 볼 수 있는 기준은 아래와 같다.

1. 사용자가 업로드 후 `두번째 시트만 읽는다는 점`을 이해할 수 있다.
2. 제목과 유의사항이 어디서 오는지 바로 확인할 수 있다.
3. 이번 달에 어떤 블록이 안내장에 들어가는지 업로드 직후 알 수 있다.
4. 미리보기와 전체 승인 흐름이 헷갈리지 않는다.
