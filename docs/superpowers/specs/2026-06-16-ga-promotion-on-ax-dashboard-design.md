# GA 프로모션 ON AX 대시보드 1차 운영설계

## 1. 목적

이 문서는 `GA 프로모션 ON AX` 대시보드의 1차 운영 방식과 데이터 흐름을 정리한 설계 문서다.  
핵심 목표는 관리자가 업로드한 엑셀을 기준으로 표준안내장을 생성하고, 최종 `전체 승인` 이후 확정 건만 사용자 화면에서 조회 및 이미지 다운로드할 수 있게 만드는 것이다.

## 2. 1차 버전 범위

### 포함

- 로그인 화면
- 대리점 프로모션 현황 화면
- 관리자 업로드 화면
- 엑셀 기반 데이터 반영
- 표준템플릿 기반 안내장 이미지 생성
- 관리자 행별 미리보기
- 관리자 `전체 승인` 1개 버튼
- 상태값 `확정 / 미확정 / 오류`
- `확정` 건만 이미지 다운로드
- 사용자 화면 상단 `최종 업로드 일시` 표시

### 제외

- PPT 생성 및 다운로드
- 행별 승인
- 수치 직접 수정
- 1차 버전의 문구 수정 기능
- DB 연동
- 실서버 배포용 권한 체계

## 3. 데이터 원본

### 3.1 템플릿 원본

- 표준안내장 원본은 `template-package.zip` 내부 템플릿을 기준으로 사용한다.
- 안내장 구조는 템플릿에 고정되어 있고, 엑셀 수치만 해당 위치에 채워 넣는다.

### 3.2 업로드 엑셀 기준

관리자가 업로드하는 운영 엑셀에서 아래 시트를 사용한다.

- `기본가동`
- `5~6월 브릿지`
- `6~7월 브릿지`
- `조직`
- `6월 지점별 담당` 계열 시트의 `GA명`

### 3.3 조직 매핑 기준

- `조직` 시트는 `GA 지사`와 `지역단 / 지점` 연결에 사용한다.
- 대시보드의 `대리점프로모션현황` 화면에서 보여주는 지역단, 지점, GA명 조회 기준도 이 매핑을 따른다.

## 4. 생성 단위

- 안내장 생성 단위는 `행 단위`다.
- 같은 `GA명`이 여러 행에 있으면 합산하지 않고 각각 별도 안내장을 만든다.
- 화면에는 `GA명만` 표시한다.
- 같은 `GA명`이 여러 건이면 목록에 같은 이름이 여러 줄 반복되어도 허용한다.
- 다운로드 파일명은 사용자에게 `GA명` 그대로 보이게 한다.
  내부 저장 시에는 충돌 방지용 숨은 식별값을 별도로 둘 수 있다.

## 5. 운영 흐름

1. 관리자가 새 엑셀 파일을 업로드한다.
2. 시스템은 맞는 행만 반영하고, 오류 행은 별도 표시한다.
3. 정상 행에 대해서만 표준안내장 초안을 생성한다.
4. 관리자만 행별 미리보기로 초안을 확인한다.
5. 관리자가 `전체 승인` 버튼 1개를 누른다.
6. 승인 결과에 따라 각 건은 `확정 / 미확정 / 오류` 상태를 가진다.
7. 사용자 화면에서는 `확정` 건만 이미지 다운로드가 가능하다.

## 6. 상태 정의

### 확정

- 필수 수치데이터가 존재한다.
- 조직 매핑이 정상이다.
- 안내장 생성이 가능하다.
- 최종 `전체 승인` 이후 다운로드 가능하다.

### 미확정

- 프로모션 수치데이터가 미입력된 경우다.
- 목록에는 보이지만 다운로드는 비활성화한다.
- 오류는 아니며, 입력 미완료 상태로 본다.

### 오류

- 데이터 반영이 불가능한 경우다.
- 목록에 보이며 다운로드는 비활성화한다.
- 오류 사유를 함께 표시한다.

예시 오류 사유:

- `조직 매핑 없음`
- `GA명 누락`
- `필수 수치 누락`
- `시트 데이터 불일치`
- `템플릿 반영 불가`

## 7. 업로드 및 반영 규칙

- 엑셀은 운영 중 계속 업데이트할 수 있다.
- 새 엑셀 업로드 시 기존 데이터는 `전체 교체` 기준으로 본다.
- 이전 생성 결과도 새 업로드 기준으로 다시 관리한다.
- 다만 업로드 즉시 확정되는 것은 아니고, 마지막 `전체 승인` 이후에만 확정 반영된다.
- 오류가 있는 파일이어도 전체 실패 처리하지 않고 `맞는 행만 반영`한다.

## 8. 안내장 생성 규칙

- 생성 시점은 `업로드 후`, `전체 승인 전`이다.
- `정상건만 생성`한다.
- `미확정 / 오류` 건은 생성하지 않는다.
- 생성 결과는 관리자만 미리보기할 수 있다.
- 1차 버전에서는 문구 수정 없이 자동 생성본 기준으로 운영한다.

## 9. 화면 설계 기준

### 9.1 로그인 화면

- 첫 화면은 로그인 페이지다.
- 사번은 `5자리 숫자`
- 비밀번호는 `동일한 5자리 사번`
- 로그인 성공 시 역할별 화면으로 이동한다.

### 9.2 사용자 화면

- 화면명: `GA대리점별 프로모션 현황`
- 조회 필터:
  - `지역단명`
  - `지점명`
  - `대리점명`
  - `상태`
- 목록 표시:
  - `GA명`
  - 지역단/지점
  - 상태
  - 다운로드
- 화면 상단에 `최종 업로드 일시`를 작게 1줄로 표시한다.
- `확정`만 다운로드 가능, `미확정 / 오류`는 비활성화한다.

### 9.3 관리자 화면

- 엑셀 업로드
- 안내장 생성
- 행별 미리보기
- `전체 승인` 1개 버튼
- 상태 요약 카드
  - 전체
  - 확정
  - 미확정
  - 오류
- 메타 정보
  - `최종 업로드 일시`
  - `안내장 생성 일시`
  - `최종 승인 일시`

## 10. 데이터 표시 규칙

- 사용자 화면에는 `GA명만` 노출한다.
- 같은 `GA명`이 여러 건이면 같은 이름이 여러 줄 반복될 수 있다.
- 내부 식별은 행 단위로 관리한다.
- 다운로드 버튼 노출 규칙:
  - `확정`: 활성화
  - `미확정`: 비활성화
  - `오류`: 비활성화 + 오류 사유 표시

## 11. 1차 버전 판단 기준

1차 버전이 완료되었다고 볼 수 있는 기준은 아래와 같다.

- 관리자가 엑셀 업로드를 할 수 있다.
- 시스템이 정상건 / 미확정 / 오류 건을 구분한다.
- 정상건만 표준안내장 이미지 초안을 만든다.
- 관리자만 초안을 행별 미리보기할 수 있다.
- `전체 승인` 후 `확정` 건만 사용자 화면에서 다운로드할 수 있다.
- 사용자 화면 상단에 `최종 업로드 일시`가 보인다.

## 12. 2차 이후 확장 항목

- 문구 수정 기능
- PPT 생성
- 승인 이력 상세 조회
- DB 저장
- Supabase 기반 사용자/권한/파일 이력 관리
- 배치 생성 자동화
