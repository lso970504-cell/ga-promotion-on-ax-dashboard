# GA 프로모션 ON AX 아키텍처

## 1. 한 줄 요약

현재 시스템은 `단일 HTML + 브라우저 메모리(localStorage) + 엑셀 업로드 + 템플릿 렌더링` 구조의 운영형 대시보드다.

## 2. 구조 요약 표

| 항목 | 내용 |
|---|---|
| Goal | GA 프로모션 현황 조회, 안내장 생성, 전체 승인, 자료 다운로드를 한 화면에서 운영한다. |
| Input | 프로모션 엑셀, 조직 시트, 권한 사번 엑셀, 기타자료실 파일 |
| Process | 엑셀 파싱 → 월별 상태 저장 → 조직 매핑 → 현황 렌더링 → 안내장 생성 → 전체 승인 → 다운로드 |
| Tools | `20260617.html`, `promo-template.html`, `template-package/promo-template.html`, `xlsx.full.min.js`, `promo-template-data.browser.js`, `localStorage` |
| Output | 프로모션 현황 대시보드, 지사별 안내장 미리보기/다운로드, 자료실 다운로드 |
| HITL | 본사담당자가 업로드, 생성, 승인 수행 |
| Failure | 시트명 불일치, 조직 매핑 실패, 확정 데이터 누락, 템플릿 데이터 누락, 브라우저 로컬 상태 꼬임 |
| Cost | 현재는 서버 비용 없이 브라우저 기반으로 운영 가능. 장기적으로는 저장소/권한 서버 필요 |

## 4. 처리 흐름

### 4-1. 로그인

1. 사용자가 사번 5자리를 입력한다.
2. 비밀번호는 같은 5자리인지 확인한다.
3. 업로드된 권한 목록과 기본 사용자 목록을 합쳐 권한을 결정한다.
4. `hq`는 관리자페이지 노출, `branch`는 조회 중심 화면만 노출한다.

### 4-2. 프로모션 업로드

1. 본사담당자가 엑셀 파일을 선택한다.
2. 브라우저에서 `XLSX`로 시트를 읽는다.
3. 단순 입력형과 레거시형을 구분해 파싱한다.
4. 월 기준으로 `promotions`, `templateForms`, `monthMetaMap` 등에 저장한다.
5. 업로드 직후 화면을 다시 렌더링한다.

### 4-3. 조직 매핑

1. 데이터 시트의 `관리지점/담당지점/지점` 값을 읽는다.
2. 조직 시트의 `지점조직명`과 매칭한다.
3. 매칭된 조직 행에서 `지역단조직명`을 가져온다.
4. 화면에는 `지역단 / 지점 / GA지사` 순서로 보여준다.

## 5. 상태 규칙

### 확정

- 업로드 데이터가 있고
- 필수 수치가 존재하며
- 템플릿 생성이 가능한 경우

### 미확정

- 프로모션 수치 데이터가 미입력인 경우
- 다운로드 비활성화 대상

### 오류

- 조직 매핑 실패
- 필수 구조 누락
- 업로드 데이터 확인 필요

## 6. 저장 방식

현재는 브라우저 `localStorage`에 저장한다.

- 로그인 권한 목록
- 월별 프로모션 목록
- 월별 메타정보
- 월별 블록 정보
- 월별 유의사항
- 월별 업로드 파일명
- 월별 승인 시각

## 7. 현재 제약

- 서버 DB가 없어 브라우저마다 저장 상태가 다를 수 있다.
- 파일 업로드 결과가 로컬 환경에 묶인다.
- 함수 중복 정의와 월별 상태 복구 로직이 복잡해 유지보수 리스크가 있다.
- 템플릿 생성과 대시보드 로직이 한 파일에 몰려 있다.
