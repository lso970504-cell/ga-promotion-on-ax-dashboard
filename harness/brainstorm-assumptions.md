## 브레인스토밍 가정 목록

### Value 가정

- 본사 운영자는 `엑셀 업로드 -> 안내장 생성 -> 전체 승인` 흐름을 가장 이해하기 쉽다고 느낀다.
- 사용자들은 사용자 수정 기능보다 `확정 여부`와 `다운로드 가능 여부`를 더 중요하게 본다.
- 사용자 화면에 `최종 업로드 일시`만 보여줘도 현재 기준 데이터를 충분히 이해할 수 있다.

### Feasibility 가정

- 표준템플릿의 고정 구조에 현재 엑셀 시트 수치를 안정적으로 매핑할 수 있다.
- `조직` 시트 기준으로 GA, 지역단, 지점 매핑이 가능하다.
- 같은 `GA명`이 여러 행이어도 행 단위 식별값으로 내부 관리가 가능하다.

### Reliability 가정

- `미확정`을 프로모션 수치 미입력 상태로 명확히 구분할 수 있다.
- 오류 행은 전체 업로드를 막지 않고도 따로 표시할 수 있다.
- 승인 전 생성된 초안과 승인 후 확정본의 상태 전환을 UI에서 혼동 없이 보여줄 수 있다.

### Viability 가정

- 1차 버전은 정적 또는 파일 기반 구조로도 충분히 검증 가능하다.
- 문구 수정, PPT 생성, DB 저장은 2차 이후로 미뤄도 운영 검증에 큰 문제가 없다.
- Supabase는 1차 필수 요소가 아니라, 운영 이력과 권한을 실제 서비스 수준으로 확장할 때 도입하면 된다.
